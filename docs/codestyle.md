# Code style
Last modified 17/11/2020

**1.** Названия классов, струкрур, и enum в `PascalCase`
**2.** Названия глобальных функций и методов в `PascalCase`
**3.** Названия аргументов в `camelCase` 
**4.** Названия локальных переменных в `camelCase`
**5.** Названия приватных полей класса `_camelCase`
**6.** Названия интерфейсов начинаются с `I`
**7.** Использование STL discouraged, нужно писать свои контейнеры. Можно использовать временно.
**8.** Документировать код в теле функции, комментарий-описание нескольких следующих строчек (пример `module_manager.cpp`)
**9.** Знак указателя на левой стороне, "соприкасаясь" с типом: `Actor*`.
**10.** Знак `{` на новой строке.
